Reports duplicate providers.

Terraform relies on plugins called providers to interact with cloud providers, SaaS providers, and other APIs.: Learn more

Example of a problem:


  provider "aws" {
    region  = "eu-west-1"
  }

  provider "aws" {
    region  = "eu-west-2"
  }

One of the ways to solve this problem is to add an alias and explicitly specify it when creating resources in the required provider.

Example of correction:


  provider "aws" {
    region  = "eu-west-1"
    profile = "default"
  }

  provider "aws" {
    region  = "eu-west-2"
    alias = "eu-west-2"
  }

  //Example of creating a resource in a specific provider. For the default provider, it's not required to specify the name.
  resource "aws_vpc" "example" {
    cidr_block = "10.0.0.0/16"
      //The default provider is used.
  }

  //For the provider alias, we specify the provider's name explicitly.
  resource "aws_vpc" "example" {
    cidr_block = "10.0.0.0/16"
    provider = "aws.eu-west-2"
  }
